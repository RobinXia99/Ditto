│                                                                                                                                                               │
│ Context                                                                                                                                                       │
│                                                                                                                                                               │
│ Ditto is a local voice-controlled AI developer assistant. Phase 1 delivers the core prototype: wake word detection → speech-to-text → command parsing →       │
│ GitHub PR summary → spoken response. All audio processing is local (no data to OpenAI). Claude (Anthropic) is the only cloud AI used.                         │

│ Key Design Decisions                                                                                                                                          │
│                                                                                                                                                               │
│ 1. Skill extensibility: Skills use a @Skill() decorator with metadata (key, description, parameters). The registry auto-discovers them. Adding a new skill =  │
│ create module + implement ISkill + import. Intent parsing prompt updates automatically from registry.                                                         │
│ 2. Silence detection for recording: Energy-based VAD (1.5s silence threshold, 10s max) instead of fixed timeout. Avoids cutting off mid-sentence or making    │
│ user wait.                                                                                                                                                    │
│ 3. Whisper fallback: whisper-node primary, shell out to whisper.cpp binary as fallback if bindings are unreliable.                                            │
│ 4. Token resolution: Keychain first → env var fallback → clear error. Setup CLI (npm run setup) for initial keychain storage.                                 │
│ 5. Single Claude client: Shared across intent parsing and analysis. Centralizes rate limiting, token counting, cost logging.                                  │
│ 6. Cost optimization: Intent parsing uses Claude Sonnet, max_tokens: 256, temperature: 0 (~$0.001/parse). PR diffs truncated to ~8000 tokens. GitHub data     │
│ cached 5 minutes.                                                                                                                                             │
